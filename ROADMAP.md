# Roadmap

## Phase 0 - Protocol Fixtures

- Finalize v0.1 terminology and schemas.
- Add valid and invalid compatibility fixtures.
- Define public-data and human-authority rules.

## Phase 1 - First Publisher

- Publish one reviewed member manifest and JSON Feed.
- Validate freshness, expiry, seeking, and offering fields.

## Phase 2 - Bootstrap Registry

- [x] Export a registry JSON document (`registry/registry.json`).
- [x] Build a human-readable directory (`directory/index.html`).
- [x] Define join via **GitHub PR** (`JOIN.md`, policy, membership PR template).
- [ ] Process first real membership PRs (opt-in publishers).
- [ ] Document update, pause, move, removal, and appeal in practice after first listings.

## Phase 3 - Agent Discovery

- Read registries and member sources.
- Produce evidence-linked recommendations.
- Require approval before contact.

## Phase 4 - Federated Pilot

- Onboard 3–5 opt-in publishers.
- Add a second compatible registry or mirror.
- Test migration, staleness, moderation, and removal.

## Later, Only if Needed

- WebFinger discovery.
- WebSub update delivery.
- Signed agent delegation.
- ActivityPub interoperability.
